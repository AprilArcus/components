Dialogs inform users about a task and can contain critical information, require decisions, or involve multiple tasks.

`Dialog` requires that the developer manages the whether the Dialog is visible or not by assigning `true` or `false` to the `isOpen` prop

```jsx
;() => {
  const { value, setOn, setOff } = useToggle()
  return (
    <div>
      <Dialog isOpen={value} onClose={() => setOff()}>
        <ModalContent>
          <Paragraph>Some content inside the Dialog</Paragraph>
          <Button>A button too!</Button>
        </ModalContent>
      </Dialog>
      <Button onClick={() => setOn()}>Open Dialog</Button>
    </div>
  )
}
```

## DialogManager

DialogManager gives an easy way to compose a Dialog without the need to manage state.

```jsx
<DialogManager
  content={
    <ModalContent>
      <Paragraph>Some content inside the Dialog</Paragraph>
      <Button>A button too!</Button>
    </ModalContent>
  }
>
  {onClick => <Button onClick={onClick}>Open Dialog </Button>}
</DialogManager>
```

## Props: width and maxWidth

All variants of Dialog allow you to override `width` and `max-width` styles to suit your content. By default, `width` is unassigned so that Dialog Surface container will conform to the width of its content. At the same time, `maxWidth` constrains the Dialog surface's width to no larger than the specified value.

`maxWidth` defaults to one of three sizes depending on the responsive breakpoint of the page (`['90vw', '90vw', '600px']`).

With those constraints in mind, if you want a variable width Dialog that has complex content you should use `maxWidth`. If your Dialog content must be a specific predefined width, assign values to both props or else `maxWidth` [will take priority](https://css-tricks.com/tale-width-max-width/).

The props accept responsive width arrays an well as static strings. Responsive array values are passed from smallest to largest breakpoints:

```jsx
<DialogManager
  content={
    <ModalContent>
      <Heading>
        Resize your browser to watch maxWidth adjust accordingly
      </Heading>
      <Paragraph>
        Lorem ipsum dolor sit amet, consectetur adipiscing elit. Nunc et eros
        sed nisi pellentesque vulputate ac eu augue. Sed commodo sagittis neque,
        vel vulputate massa.
      </Paragraph>
    </ModalContent>
  }
  maxWidth={['90vw', '60vw', '500px', '800px']}
>
  {open => <Button onClick={open}>Open responsive width modal</Button>}
</DialogManager>
```
